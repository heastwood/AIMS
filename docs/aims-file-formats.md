# AIMS File Formats

## JAF — Job Allocation File

The JAF is the **input checklist** — one record per mailpiece, generated by OMS
before the run begins. It is the Quadient equivalent of an industry MRDF (Mail
Run Data File). See `aims-mrdf-reference.md` for the terminology mapping.

### Key Design Principles

- **10-digit JobID + 10-digit MailPieceID** uniquely identify every record
- **PHI and PCI are NOT included** — AIMS does not store sensitive recipient data unless explicitly provided
- OMS (Inspire/Automate) holds PII; JAF carries only IDs and machine control instructions
- Dropped into the AIMS watched folder (FileDrop or jaf-input) for database ingestion

### Simulator-Generated JAF (AIMS360 JSON format)

When using the DS Simulator, the JAF takes the form of a `.json` file placed in the FileDrop folder:

```json
{
  "jobId": "31CD77GGBW",
  "mailPieceCount": 25,
  "imosName": "IMOS_006",
  "imosNameId": "SIMU"
}
```

> ⚠️ `"imosNameId": "SIMU"` indicates a simulator-generated profile.
> The machine config in AIMS must map `IMOS_006` to a registered machine.

## JCF — Job Complete File

The JCF is the **output audit file** — generated at job completion. It is a
fixed-width flat file with one header line and one body line per mailpiece.

### JCF Field Layout

*Reverse-engineered from sample file `5450D6E900.jcf`*

#### Header Line (marker = `H`)

| Python Slice | Length | Field |
|--------------|--------|-------|
| `[0]` | 1 | Record marker: `H` |
| `[1:11]` | 10 | Job ID |
| `[11:30]` | 19 | Padding / blank |
| `[30:49]` | 19 | Job start timestamp (`YYYY-MM-DD HH:MM:SS`) |

#### Body Line (marker = `B`, one per mailpiece)

| Python Slice | Length | Field |
|--------------|--------|-------|
| `[0]` | 1 | Record marker: `B` |
| `[1:11]` | 10 | Job ID |
| `[11:21]` | 10 | Piece ID (MailPieceID) |
| `[21:23]` | 2 | Piece document count |
| `[23:42]` | 19 | Piece completion timestamp (`YYYY-MM-DD HH:MM:SS`) |
| `[42:50]` | 8 | Piece hash / checksum |
| `[50:75]` | 25 | Reserved / unknown |
| `[75:121]` | 46 | Counter block (feeder/insert counts) |
| `[121:171]` | 50 | Full name |
| `[171:221]` | 50 | Organization |
| `[221:271]` | 50 | Address Line 1 |
| `[271:321]` | 50 | Address Line 2 |
| `[321:371]` | 50 | Address Line 3 |
| `[371:421]` | 50 | Address Line 4 |
| `[421:471]` | 50 | Postal / State + ZIP |
| `[471:621]` | 150 | Reserved / blank |
| `[621]` | 1 | Divert / status flag |

### JCF Disposition Codes

| Code | Meaning |
|------|---------|
| `00` | Successfully processed |
| `01` | Diverted / not inserted |
| `02` | Reprint required |
| Other | Refer to AIMS documentation for extended codes |

### Python Parser — 1-based field extraction helper

```python
def f(line, start, length):
    """Extract field using 1-based spec positions (strips whitespace)."""
    return line[start - 1 : start - 1 + length].strip()

# Header
job_id          = f(line,  2, 10)   # pos 2, len 10
job_sla_dt      = f(line, 12, 19)   # pos 12, len 19
job_creation_dt = f(line, 31, 19)   # pos 31, len 19

# Body
mail_piece_id   = f(line, 12, 10)   # pos 12, len 10
disp_code       = f(line, 22,  2)   # pos 22, len 2
result_ts       = f(line, 24, 19)   # pos 24, len 19
customer_id     = f(line, 43, 32)   # pos 43, len 32
```

## JRF — Job Reprint File

Generated automatically when AIMS detects missing or failed pieces. Contains
the subset of pieces requiring reprinting.

- Shares the DocSecure `export` folder path with the JCF
- Format mirrors the JAF — same record-per-piece structure for reprint jobs
- AIMS creates reprint files; they must be processed through OMS like a new run

## Output CSV Columns (from jcf_to_csv.py)

The JCF parser script outputs these columns to `aims_jobs.csv`:

```
source_file, job_id, job_creation_datetime, mail_piece_id,
disposition_code, disposition_description, result_timestamp,
customer_id, total_prime_documents,
feeder_1_count, feeder_2_count, ..., feeder_16_count,
envelope_divert_1, envelope_divert_2, envelope_divert_3,
form_divert_1, form_divert_2, franker_1, franker_2,
halt, ink_mark_1_jog, ink_mark_2_pulse, no_seal, auto_end,
full_name, organization,
address_line_1, address_line_2, address_line_3, address_line_4,
postal_code, user_field_1, user_field_2, user_field_3,
full_format_pack_divert, parsed_at
```

> ⚠️ The `customer_id` field may contain a hash/checksum value rather than a
> traditional ID in some AIMS configurations. Verify with a real production JCF.
